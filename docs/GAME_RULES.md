# 게임 규칙과 조작

## 공통

- 액션 게임은 시작 전에 카운트다운을 표시하며 공통 HUD에서 일시정지·다시 시작이 가능하다.
- 탭이 숨겨지면 자동 일시정지되고 사용자가 직접 계속하기를 눌러야 한다.
- 이름은 설정에서 바꾸며 비어 있으면 `PLAYER 1`, `PLAYER 2`를 사용한다.
- 진동과 전체 화면은 지원하는 브라우저에서만 보조적으로 사용한다.

## 오목

15×15 교차점에 흑부터 번갈아 돌을 둔다. 가로, 세로, 두 대각선 중 같은 돌 5개 이상이 연속되면 승리한다. 이미 놓인 곳, 보드 밖, 종료 후 입력은 무시한다. 마지막 수만 무를 수 있으며 무르기 전에 확인한다. 승리가 확정된 뒤에는 무를 수 없다. 기본은 3판 2선승이고 단판과 5판 3선승을 선택할 수 있다. 선공 교대를 켜면 새 라운드마다 흑을 맡는 플레이어가 바뀐다. 마우스·fine pointer는 교차점에 즉시 착수하고, 터치·coarse pointer는 첫 터치로 좌표를 선택한 뒤 44px 이상 확정 버튼으로 착수한다. 선택 변경·취소와 좌표 select·키보드 조작도 제공한다.

## 네온 탁구

P1은 `W/S`, P2는 `↑/↓`로 움직인다. 모바일은 좌우 플레이어의 상·하 버튼이 분리된다. 패들에서 맞은 위치와 이동 방향이 반사각에 반영되고 랠리가 길수록 제한 범위 안에서 빨라진다. 득점 후 짧은 재서브를 거친다. 기본 7점, 5/7/11점 선택이다.

## 통통 배구

P1은 `A/D/W`, P2는 `←/→/↑`를 사용한다. 모바일은 각자의 좌·우·점프 버튼을 사용한다. 선수는 자기 코트와 네트 경계를 넘지 못한다. 공이 바닥에 처음 닿는 프레임에만 상대가 득점하고 선수와 공을 재배치한 뒤 카운트다운한다. 기본 7점, 5/7/11점 선택이다.

## 핀볼 늦게 떨어지기

같은 seed로 만든 대칭 레인에서 두 공이 동시에 떨어진다. P1은 `A`, P2는 `L` 또는 각자의 부스터 버튼을 사용한다. 라운드당 세 번, 짧은 쿨다운 뒤에만 위쪽 힘을 줄 수 있다. 공 하단이 바닥선을 처음 넘은 시간을 기록한다. 차이가 0.05초 이하면 무승부이며 기본 매치는 3판 2선승이다.

## 사다리 타기

2–8명의 이름과 결과를 입력한다. 빈 항목은 안전한 기본값으로 바뀐다. 브라우저 보안 난수로 인원수 `n`의 `n!` 순열 중 한 개를 선택하며 제자리 결과도 허용한다. 선택한 순열은 한 행에 하나의 인접 교환으로 정확히 구성하고, 역계산한 mapping이 다르면 생성을 중단한다. 생성 시점의 이름·결과·가로줄·mapping·라운드 ID를 snapshot으로 고정한다. 결과 확인 중에는 설정을 잠그고, 편집으로 돌아가면 확인 후 snapshot과 공개 상태를 폐기한다. 한 명씩 경로를 공개하거나 전체 결과를 볼 수 있으며, 전체 공개 전에는 전체 mapping·라운드 ID·복사를 제공하지 않는다.

## 반응속도 대결

P1은 `F`, P2는 `J` 또는 화면의 각자 영역을 누른다. 매 라운드 예측하기 어려운 시간이 지난 뒤 신호가 켜지며 먼저 누른 사람이 이긴다. 신호 전에 누르면 부정 출발 패배다. DOM event timestamp를 performance clock으로 정규화한다. 두 입력을 12ms 동안 수집하고 신호 기준 반응 시간의 차이가 8ms 이하면 동점으로 처리한다. 포인터 ID 중복과 compatibility click은 한 번만 판정하며 먼저 3점을 얻은 플레이어가 승리한다.

## 탭 배틀

P1은 `F`, P2는 `J` 또는 각자 화면 영역을 연타한다. 5/10/15초를 고를 수 있고 기본은 10초다. 키 자동 반복, 동일 pointer의 중복 down, compatibility click은 집계하지 않는다. 유효한 입력은 모두 점수에 반영하되 hit 사운드는 40ms 이상의 간격을 두어 고속 입력이 Web Audio를 과도하게 호출하지 않게 한다. 실시간 탭 수, 초당 탭 수, 우세 게이지를 표시하며 종료 시 최종 수가 높은 플레이어가 승리한다.

## 벌칙 룰렛

2–12개의 항목을 입력한다. 빈 항목은 번호가 붙은 기본 항목으로 대체하고 긴 문자열은 안전하게 줄여 표시한다. 모든 칸은 같은 확률이다. 시작할 때 결과 index를 먼저 고르고 애니메이션의 최종 각도가 그 칸의 중심과 정확히 일치한다. 회전 중에는 다시 돌리기와 편집을 잠그며 결과를 클립보드에 복사할 수 있다.
